Forms

Useful tools!!!

In addition to text and images, forms are another common web feature. Often they are used with an external service (like a search box), but we can start to practice with it by passing things to our local scripts.

form and input are the main tags we'll use here. form goes around the input like so : 

<form>
        <input type="text" /> 
        <input type="submit" /> 
</form>

to make soemthing like this functional, we need to add a bit more.

<form action="login.php" method="POST" name="sampleForm">

looks more like it. The attributed do the real work here. action will contain a script or file to process the form data. here, it's calling a PHP page (php gets executed on the server). method is genrally either a GET or POST request.

GET submits the data as a URL query, it might look like this : https://www.google.com/search?q=hellraiser. For legibility, that's not too bad, however we're sort of screwed if we want to pass a username+login this way : http://mysite.com/login.js?user=Mark&pass=admin. This makes everything viewable in the browser. While GET requests are great for retreiving data (like from APIs), they're less good at sending information.

POST requests submit data as part of the body of the request, so it's not blatantly visible when you're sending a request.

The name attribute is helpful when you have multiple forms on a page or site. it's really helpful when dealing with server side programming, but it's good practive to name forms regardless.
