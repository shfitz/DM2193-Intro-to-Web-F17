Accessing the DOM (Document Object model)

The DOM is a programmig interface to HTML. It's the way we can start to do fun shit with javascript and other things in our sites. Through the DOM, we can treat HTML documents like objects.

  • Everything is a node (including the document itself)
  • all HTML elements are 'element nodes'
  • all HTML attributes are 'attribute nodes'
  • text indise elemts are 'text nodes'
  • hell, even comments are nodes called (surprise!) 'comment nodes'

When you write HTML, you aren't writing the DOM, it is created when the page loads into your browser. The page is the 'owner' of all the elements, it's considered the root node. All the nodes can be accessed through javascript, but you have to navigate to the node you want to change. Things get weird in that we can create, modify and destroy elements, CSS classes, pretty much anything on the page.
