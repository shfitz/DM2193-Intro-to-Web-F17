Functions

functions are repeatable blocks of code that you can name.

Syntax

There's a keyword function that identifies when you're creating a new one. Don;t cram too much stuff into them. Use meaningful (and hopefully short) names to identify what task the fucntion is executing.

function myFucntion(argument0, argument1){
    // do something here
    }

You use a function by invoking it. That is, you call it by name in your program. Thsi is doens usualling in another fucntion or when an event happens.

Events

onclick

javascript is driven by events. On the web, there are a bunch of different events that happen in a page. onload, onclick, hover. When we want to leverage an event, we tie a fucntion to an event listener. This listens for somethig to happen, then executes our code. For example, we can use a button to do some math for us.

onload

With great power comes great responsibility, and this means you have to decide what you do when pages load. Are you going to throw up some arbitrary popup window just because you can? (hint : today you are)
