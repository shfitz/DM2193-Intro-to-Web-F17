Geolocation

You'll need to be running this page on a server for it to work (the file:// protocol doesn't support geolocation).

the geolocation API is a javascrip object that is part of the window.navigator object. you access it by calling window.navigator.geolocation

to make sure the browser you're using supports geolocation, you can use this function : 

function getMyLocation() {
 if (navigator.geolocation) {
  // you support location!
  alert("I can see you");
 } else {
  alert("Geolocation is not supported by this browser.");
 }
}

If your browser does support this, you can call the getCurrentPosition() method. This will ask the user if they give permission for the browser to share location information. This is not available immesiately, it's asynchronous. that is, we need to wait for it to happen. Because of this, it's useful to put this in a callbak function.

// if getCurrentPosition is successful, it calls the function gotPosition, which is passed a javascript object wth information about our location
function getThisLocation() {
 if (window.navigator.geolocation) {
  window.navigator.geolocation.getCurrentPosition(gotPosition);
} else {
  alert("Geolocation is not supported by this browser.");
 }
}

function gotPosition(position) {
  // print location to the console
  console.log(position);
}

the coords object has some information for us, specifically the latitude and longitude. We can stick this in our page like so : 

function showThisLocation() {
 if (window.navigator.geolocation) {
  window.navigator.geolocation.getCurrentPosition(showPosition);
} else {
  alert("Geolocation is not supported by this browser.");
 }
}

function showPosition(position) {
  var lat;
  var lon;
  lat = position.coords.latitude;
  lon = position.coords.longitude;
  document.getElementById("dispLoc").innerHTML="latitude: "+lat+", longitude : " +lon;
}
